# Contributing to AspireKeyCloakTemplate

Thank you for your interest in contributing! Please follow these guidelines to help us maintain a high-quality codebase.

## Project Structure
- AppHost: Application entry point and orchestration
- AspireKeyCloakTemplate.API: Main API logic
- AspireKeyCloakTemplate.Gateway: API Gateway and BFF
- AspireKeyCloakTemplate.ServiceDefaults: Shared service configuration
- react-app: Frontend UI

## Contribution Workflow
1. Fork the repository and create a feature branch
2. Follow code style and naming conventions (see .editorconfig, NAMING_CONVENTIONS.md)
3. Write tests using Shouldly, NSubstitute, Vitest, and Playwright as appropriate
4. Ensure all tests pass before submitting a PR
5. Reference relevant documentation in your PR description

## Documentation
- See .copilot-instructions.md for architecture and best practices
- See NAMING_CONVENTIONS.md, ERROR_HANDLING.md, CACHING_STRATEGY.md, TESTING_GUIDELINES.md for detailed guidelines

## Code of Conduct
- Be respectful and constructive
- Review and respond to feedback promptly
